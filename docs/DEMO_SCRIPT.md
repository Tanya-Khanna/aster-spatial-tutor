# Aster✱ — 2:45 demo script

The video should prove one memorable claim: **Aster✱ turns the learner’s current screen into the teaching surface.** Keep the cursor visible, use the native app—not a concept mockup—and show the rubric evidence on screen.

## 0:00–0:18 — Problem

Show a dense research-paper equation in a normal Mac window.

> “When a research paper, anatomy plate, or derivation loses you, today’s workflow is: leave the page, take a screenshot, upload it, describe where you’re confused, then map paragraphs of chat back to the source. The hardest part becomes translating the page.”

## 0:18–0:33 — Promise

Launch Aster✱ and briefly show its welcome screen.

> “Aster✱ is a spatial AI tutor for macOS. It sees one screen state when invited, listens to your question, and teaches by voice while drawing directly where the idea lives.”

## 0:33–1:18 — Hero research-paper flow

1. Press **⌥ Space** and drag tightly around the equation.
2. Show the visible local Follow state.
3. Ask: “Why are we dividing by the square root of the key dimension?”
4. Choose Aster✱’s diagnostic option: “Why softmax needs scaling.”
5. Let the overlay reveal two synchronized teaching steps while Aster✱ narrates.
6. Answer the independent transfer question.
7. Show the assessment and the persisted memory: understood softmax purpose; scaling still shaky—or mastered if answered correctly.

> “Aster✱ first separates likely misconceptions instead of guessing. GPT-5.6 then returns a schema-constrained, staged lesson—not free-form drawing commands. Voice carries the explanation, the overlay carries the reference, and a separate assessment updates evidence about what this learner actually understands.”

## 1:18–1:48 — Breadth without losing the story

Use Demo mode to switch to Anatomy, then Math. Do not run three full lessons.

> “The interaction generalizes to visual material where prose alone is weak: trace blood flow through a heart, pair a function with its derivative, or explain the role of a term in a circuit. Aster✱ stays above the learner’s existing app instead of forcing an upload workflow.”

## 1:48–2:15 — Agent restraint

Show the “Show in graph sandbox” preview row.

> “When seeing a change matters, Aster✱ opens a real embedded Desmos sandbox or one of its bounded Manim animation families. Actions are permissioned and reversible; Aster✱ never executes model-authored code or modifies graded work.”

## 2:15–2:35 — Technical proof and budget

Quickly show the README architecture diagram and the local budget line in the app.

> “The app is native SwiftUI and AppKit: exact context cropping, Retina coordinate mapping, local follow mode, a click-through overlay, native speech, Keychain, persistent concept memory, and three strict Responses API contracts. Terra handles vision, Luna checks answers, and Sol is an explicit precision mode. The deterministic demo costs zero.”

## 2:35–2:45 — Close

Return to the annotated page.

> “Aster✱. Your screen becomes the whiteboard.”

## Recording checklist

- 16:9, 1080p, no notifications, cursor large enough to see.
- Keep the final video below 3:00.
- Demonstrate an actual packaged build.
- If using a live API call, rehearse once and keep Demo mode ready as fallback.
- Show the capture permission prompt only if it helps the privacy story.
- Add captions; the product voice must not compete with narration.
- End card: product name, Education Track, repository URL, landing-page URL.
